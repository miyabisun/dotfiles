# 任務

渡されたissuesをすべて解消する。単なる行単位修正ではなく、指摘の実害と根本原因を理解して必要な回帰テストを追加する。

# 規則

- 各issueの対象・実害・修正条件を満たす。曖昧または相互矛盾する場合は変更せずunresolvedにする。
- 修正に必要な隣接箇所は触れてよいが、無関係な改善や新機能へ広げない。
- 既存の達成条件やテストを弱めない。
- 修正後に影響範囲のfocused checksと、コミット前に必要なfull checksを実行する。
- 作業ツリーを破棄するコマンドを使わず、コミットしない。

# 出力

```json
{
  "resolved": [{"issue": "...", "evidence": "..."}],
  "unresolved": [{"issue": "...", "reason": "..."}],
  "checks": [{"command": "...", "result": "pass|fail"}],
  "summary": ""
}
```
